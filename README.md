# California Housing — Price Model Dashboard

A small but realistic ML project: an **XGBoost** regression model (with
**Bayesian hyperparameter tuning** via Optuna) served behind a **Streamlit**
dashboard.

It doubles as the live demo for a talk on **`git worktree`** — training/tuning
takes long enough that you genuinely can't just switch branches in the same
working directory while it runs, which is the whole point. The slides and the
step-by-step demo script live in [`presentation/`](presentation/).

## Quick start

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt

# 1) train a model (writes models/model.json + models/metrics.json)
python -m backend.train --mode tuned       # Bayesian search — the slow one
#   or, for a quick baseline:
python -m backend.train --mode baseline

# 2) launch the dashboard (reads the artifacts above)
streamlit run frontend/app.py
```

Tip: a second instance on another port — `streamlit run frontend/app.py
--server.port 8502` — is handy for previewing a change made in a worktree.

## Layout

```
backend/
  data.py     load California housing + engineer features (cached after first fetch)
  train.py    fit the model; --mode {baseline,tuned}; writes models/ atomically
  tuning.py   Optuna Bayesian (TPE) hyperparameter search  <- the long job
  model.py    load the trained model + metrics, run predictions
frontend/
  app.py      Streamlit dashboard (Plotly charts)
  theme.py    chart colour palette  <- single source of truth for colours
  whatif.py   interactive what-if price predictor panel
notebooks/
  01_eda.ipynb               exploratory data analysis
  02_model_experiments.ipynb baseline vs Bayesian-tuned experiment (the long job)
tests/        pytest suite
presentation/ the git-worktree talk: slides + demo runsheet
models/       trained artifacts (gitignored — regenerated by train.py)
```

## Model

- **Data:** scikit-learn's California Housing (20,640 rows, 8 features), plus two
  engineered ratios (`bedrooms_per_room`, `rooms_per_person`).
- **Model:** `xgboost.XGBRegressor`.
- **Tuning:** Optuna TPE sampler over a 5-fold cross-validated RMSE objective.
  Trial count is set by `--trials` (or the `N_TRIALS` env var). Bump it to make
  training take longer.
